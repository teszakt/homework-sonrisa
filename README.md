NOTIFICATION ABSTRACTION
------------------------
For simplicity users, events and alerts are created through REST endpoints.
Later events and alerts could be handled in an event-driven way (using MQ technology like RabbitMQ or Kafka).

abstract notification for future extension
abstract NotificationSender
NotificationChannelType [EMAIL, SLACK]
other channels (later like SMS, Teams, etc)

NotificationSender (abtract)
send() abstract method will be implemented for each of the notification channel
EmailNotificationSender
SlackNotificationSender
etc. for any other channel NotificationChannelType

NotificationDispatcher
for alers in alert.channels (through channel field in Alert entity)
find sender (User)
then sender.send()

EventService interface (register events then find matching alerts and dispatch notifications)
then
AlertEvaluatorService (evaluates incoming Event based on existing Alerts)
then
NotificationService notify
then
NotificationDispatcher
